HTML基础(一):文档模式

DOCTYPE

DOCTYPE (Document Type Declaration - 文档类型声明)
通常位于一个HTML文档的开头，在HTML5中通常写成<!DOCTYPE html>用于在文档解析一开始就让浏览器知道该文档用的文档模式

现有的文档模式

在最初各大浏览器厂商定制规则时，定制了两种模式: 标准模式 和 混杂模式
标准模式是为了解决web开发时各大浏览器间兼容的问题制定的标准,遵循W3C标准(IE6开始使用,也叫严格模式)
混杂模式是为了保持以前开发的web项目依然可以兼容而制定的标准,遵循浏览器自身的标准(也称为怪异模式)

后来IE又提出了一种模式叫准标准模式,与标准模式几乎没有区别，只有在处理图片间隙的时候会有一定区别

各种类型DOCTYPE的写法

后面两种过渡型和框架集型即为准标准模式的触发方式
想触发混杂模式,只要不写DOCTYPE或者写错就行

特殊情况
如果在写代码时在DOCTYPE声明之前添加了以下内容: 普通文本、HTML 标签、HTML 注释、XML 声明、IE条件注释,
其中如果是普通文本或HTML标签,所有浏览器都会进入混杂模式,
而如果是HTML注释或XML声明，在非IE浏览器中会被忽略，而在IE中，如果是HTML注释则所有IE浏览器都会进入混杂模式，而xml只有在IE6会进入混杂模式，
IE条件注释：这个是唯一在所有浏览器中都不会进入混杂模式的方法，所以要是非得在HTML前面写点啥，一定要写在以下标签中

检测文档模式

想要知道文档用了哪种模式可以通过document.compatMode的返回值来判断

名称 英文名称 返回值
标准模式 standards mode/strict mode CSS1Compat
准标准模式 almost standards mode CSS1Compat
怪异模式 quirks mode BackCompat

标准模式和混杂模式的区别

盒模型

标准模式下的盒模型宽度width为content的宽度
混杂模式下的盒模型宽度width为border + padding + content

在标准模式下使用box-sizing: border-box会使盒模型计算方式和混杂模式一样

行内元素的对齐方式

标准模式下对齐至基线
混杂模式下对齐至底部

!important声明

混杂模式下，IE6/7/8都不认!important声明

行内元素宽高

标准模式下行内元素直接设置width和height都不会生效
混杂模式下直接生效

标准模式下需要设置display: inline-block才可以设置宽高

水平居中

标准模式下使用margin: 0 auto会使元素水平居中
混杂模式失效
