JS基础(一):在HTML中使用JS

使用<script>标签

  • <script>的属性
属性名 是否必选 作用
async 否 表示需要立即下载脚本，但不妨碍页面中其他操作（异步下载）
defer 否 表示脚本可延迟到文档被解析后再执行
src 否 表示外部脚本的路径
charset 否 表示通过src指定的代码的字符集
language 否（已废弃） 表示编写代码使用的语言
type 否 可以看成language的替代属性，表示MIME类型

1.async与defer

先看以下三种引入JS脚本的方式

使用第一种方式时浏览器读到script标签就会立即开始加载并执行该脚本，同时暂停解析后续HTML文档，直到脚本执行完才会恢复HTML文档解析;
使用第二种方式时浏览器会同时进行脚本加载和HTML的解析，但在脚本加载完成的同时会暂停HTML的解析并开始执行脚本，脚本执行结束恢复解析HTML(使用async方式不保证脚本按序执行，所以使用时要注意脚本间不要存在依赖关系);
使用第三种方式时浏览器会同时进行HTML的解析和脚本加载，HTML解析完成后才按顺序执行脚本;

2.src属性

src属性表示脚本的路径，可以填成绝对路径(如：src="https://cdn.bootcss.com/jquery/3.3.1/core.js")，也可以填成相对路径(如：src="../js/example.js")，根据具体情况使用即可;

3.type属性

type属性表示脚本的MIME(内容类型)，默认为text/javascript而实际在服务器传送JavaScript脚本时MIME类型为application/x-javascript，在非IE浏览器中还可以使用application/javascript和application/ecmascript。一般情况下使用默认值即可;

<script>放置位置

传统的做法把引入的css文件和脚本文件放置在<head></head>中，但这种做法会导致必须下载完所有脚本后才能解析dom，所以最好把所有的script标签都放在<body></body>的最后;
由于不同浏览器对于defer属性支持不同，因此哪怕是defer脚本也最好放在<body></body>的最后;
